What is this?

SQL Injection is a way for people to input specific SQL statements to a database, bypassing very basic security checks and possibly revealing sensitive information. Features on sites such as login pages, feedback and support forms, and shopping carts can all be pages targeted by someone. This comic on xkcd gives a basic example of this idea.

One simple way to prevent this is by using an input validation technique called prepared statements, or parameterized queries. This method allows the database to distinguish between code and data, and can prevent a user adding an extra condition to a SELECT statement to grab more records than allowed. There are a few other options, but the concensus seems to be that this is the first method one should be taught, and advanced techniques combine it and others.
