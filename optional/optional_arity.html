Arity with Optional Parameters

Home About Extras

In recent code I've written in Ruby I've started to include assert statements to ensure that certain conditions are met. One method I've learned that I now use frequently in my asserts is "arity," which returns the number of parameters a method takes. Here's one example of its use:

def Object
def initialize(name)
@name = name
end
end

assert(Object.method(:initialize).arity == 1)

In this case the assert function will pass, as the initialize method only takes one parameter. What would #arity return if the method had optional parameters though?

def add(*items)
items.each{|item| @list << item}
end

assert(Object.method(:add).arity == 0)
assert(Object.method(:add).arity == 1)
# Both of these asserts will fail.

Ruby handles this in a somewhat odd way. To represent the arity of a method with optional parameters, Ruby takes the number of required parameters, makes it negative, then subtracts one from that number. If n represents the amount of required parameters, the arity of a method can be represented by (-n - 1). In the previous example, there are 0 required parameters, so (-0 - 1) = -1, which is the arity of :add.

def add(*items)
items.each{|item| @list << item}
end

assert(Object.method(:add).arity == -1)
# This assert will pass.
